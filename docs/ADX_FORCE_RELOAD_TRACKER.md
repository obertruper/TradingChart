# ADX Force Reload Tracker

## Причина пересчёта

Коммит `5b9767c` (2026-02-24) — переход ADX на single-pass архитектуру (OBV-паттерн).
Старая логика ломала цепочку Wilder smoothing на границах батчей, что давало ~66% точность.
Новая логика загружает ВСЕ свечи, считает TR/DM один раз и применяет Wilder smoothing на полной цепочке.

**Все данные ADX, рассчитанные ДО 2026-02-24, содержат ошибки и требуют `--force-reload`.**

## Команда для пересчёта

```bash
cd /root/TradingCharts/indicators

# Конкретный символ + таймфрейм
python3 adx_loader.py --symbol BTCUSDT --timeframe 1m --force-reload

# Конкретный символ, все таймфреймы
python3 adx_loader.py --symbol BTCUSDT --force-reload

# Конкретный таймфрейм, все символы
python3 adx_loader.py --timeframe 1m --force-reload
```

Данные практически не изменились. Старый batch-алгоритм на практике давал корректные
результаты для ADX, так как загружал данные с достаточным lookback и цепочка Wilder
smoothing не ломалась на batch-границах.

Тем не менее, пересчёт полезен:
- Данные теперь **гарантированно** корректны (single-pass через всю историю)
- Новый алгоритм **быстрее** (все 8 периодов за один проход vs 8 отдельных)
- **Надёжнее** — нет зависимости от размера batch и lookback
